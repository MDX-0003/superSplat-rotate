## 需求分析

### 需求 1: Fuse PLYs 滚动

与已实现的 Render PLYs 滚动一致：包裹 `render-table-wrap` div，CSS 复用已有类。排序规则不变（按文件名字母序）。

### 需求 2: 自动选中最新渲染 PLY

当前流程问题：
```
点击 "interpolate + fuse + clip"
  → doFuse() → POST /fuse → 200 OK → location.reload()
  → 后台线程 run_fuse_clip() 开始执行
  → ... interpolate → fuse → clip ...
  → SSE broadcast "status" → 浏览器 location.reload()
  → 新 PLY 出现在列表中，但没有任何选中
```

解决方案：
- `run_fuse_clip()` 完成后，将新生成的 PLY 名称存入 `state.last_clipped_ply`
- `build_fuse_page()` 读取该字段，在 HTML 中 pre-check 对应的 radio 按钮
- 消费后清除标记（避免后续普通刷新重复选中）

### 需求 3: JSON 单选记忆

使用 `localStorage` 存储用户选中的 JSON 文件路径。页面加载时恢复。行点击时更新存储。

**关键设计点**：HTML radio 的 `value` 是 index，但 index 在刷新后可能变化（新 JSON 插入）。因此用 `data-path`（文件完整路径）作为稳定标识。

    function selectOne(tr) {{
      let radio = tr.querySelector('input[type="radio"]');
      radio.checked = true;
      let col = tr.dataset.col;
      for (let r of document.querySelectorAll('tr[data-col="' + col + '"]')) {{

## 讨论要点

有几个设计选择想确认：

### 1. `last_clipped_ply` 的消费时机

当前设计：`build_fuse_page()` 读取 → 立即清空。这意味着只有 fuse+clip 完成后**下一次**页面加载才会自动选中。

如果用户在 fuse+clip 完成前手动刷新了页面（比如 F5），`last_clipped_ply` 还是上一次的值，会在那次加载中被消费掉。这种情况很少见，应该可以接受。

### 2. JSON 记忆的 key

使用完整路径（`data-path`）作为 localStorage 的标识。不同项目的路径不同（如 `E:\work\...\CameraData\05\cameras.json`），不会冲突。但如果 JSON 文件被删除后又有同名文件出现，会被自动恢复选中——这通常是期望行为。

### 3. 是否也在 Render PLY 上做 localStorage 记忆？

目前只对 JSON 做记忆。如果用户希望 Render PLY 选中也被记忆，可以同模式追加。但需求中只提到了 JSON，这次不做。

### 4. auto-select 的范围

自动选中仅在 fuse+clip 成功时触发。如果 fuse 失败（clip 没执行），不会触发自动选中。超时或异常也不会。这符合"新生成的"语义。
