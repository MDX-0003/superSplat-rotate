## 一、工作概述

## 二、SuperSplat Web 应用改动

### 2.1 批量导出 GT 相机渲染

**功能入口**：View Panel → GT Camera 区域 → **Export Renders** 区域

**UI 控件**（`src/ui/view-panel.ts`）：

| 控件 | 参数 | 默认值 | 说明 |
|------|------|--------|------|
| Export All 按钮 | — | — | 触发批量导出 |
| Resolution | SelectInput | HD (1920×1080) | viewport / HD / QHD / 4K / custom |
| Width / Height | NumericInput | 1920 / 1080 | custom 模式显示 |
| Max Poses | NumericInput | 44 | 只导出前 0~X 号位姿 |
| Circle Center | Label (只读) | 自动计算 | 质心显示 |
| Offset Mode | SelectInput | towardCenter | towardCenter / alongForward |
| Offset | NumericInput | 0 | 偏移距离 |

**渲染逻辑**（`src/render.ts`，`render.batchGtCameras` 事件函数）：
- 进入 offscreen 模式一次（避免反复重建 RT）
- 循环每个 pose：attch sorter 监听 → fire `camera.setPose` → 等待 sort → `setTargetSizeOverride` → `forceRender` → `await postRender()` → `copyRt` → `read` → `PngCompressor` → `downloadFile`
- 每帧分配新 `Uint8Array`（防止 PngCompressor Web Worker 转移 buffer 导致 detached）
- `forceRender = false` 清除残留标志 → 防止首帧重复
- 支持 progress bar + 取消

**偏移逻辑**（`applyPoseOffset` in view-panel.ts）：
- `towardCenter`：`direction = normalize(center - position)`
- `alongForward`：`direction = rotation.transformVector([0,0,-1])`
- Apply 按钮预览和 Export All 共用同一函数

### 4.3 关键算法

**圆拟合**（`fit_circle`）：
1. SVD 分解所有相机位置 → 最佳拟合平面（`u1, u2, normal`）
2. 投影到 2D 平面 → 最小二乘圆拟合 → 圆心 + 半径
3. 返回 3D 圆心 + 平面基向量

**旋转生成**（`interpolate_cameras_circle.py`）：
1. 对每个采样位置计算 `lookAt(center)` 的 COLMAP 旋转矩阵
2. 计算锚点相机的实际旋转与 look-at 的残差
3. 在 360° 角度域对残差做 slerp
4. 最终旋转 = `lookAt * slerp(residual)`

**半径插值**（椭圆弧）：
- 两锚点半径 r_a, r_b 在角度域线性插值
- 两个"半圆"各占不同比例，边界平滑过渡

[0] 原始采集 (手动/硬件)
        │

## 八、SuperSplat 手动操作步骤（Step 4 详细流程）

1. 浏览器打开 SuperSplat
2. 拖入项目中的 `.ply` 点云模型
3. File → Import → 选择 `cameras_align.json`
4. 右侧工具栏点击相机图标（GT Camera）→ View Panel 自动打开
5. 在 GT Camera 区域配置：
   - Resolution: HD (1920×1080)
   - Max Poses: 自动
   - Offset Mode / Offset: 按需
6. 点击 **Export All**
7. 浏览器自动下载 300 张 `circle_XXXX.png` 到 Downloads
8. 将全部 `circle_0*.png` 移入项目的 `renders/` 目录
9. 终端按 Enter 继续
